# TODO

Backlog of follow-up work. New items get added here as they come up during
development; completed items are removed (git history is the audit trail).

## Conventions

**Priority** (urgency, not difficulty):

- **P0** — blocker, user-facing breakage, or data-loss risk. Drop other work.
- **P1** — important; schedule into the next working session.
- **P2** — nice-to-have, polish, or exploratory.

**Format:**

```
- [ ] **P1** Short imperative description — optional context / why.
```

Use `- [~]` for in-progress, `- [x]` only briefly before deleting the line.
Order tasks within each epic by priority (P0 first), then by date added.
